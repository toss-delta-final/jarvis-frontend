# Jarvis 페이지별 목적 · 핵심 기능 정리

---

## 1. 로그인 / 회원가입

**목적**
- 구매, 개인화 추천, 채팅 횟수 상향 등 회원 전용 기능을 위한 인증 진입점
- 비로그인(게스트)도 탐색·추천은 가능하므로, 로그인을 강제하지 않는 선택적 진입 화면

**핵심 기능**
- 이메일 + 비밀번호 로그인 / 회원가입 (닉네임, 약관 동의 포함)
- 이메일 중복 확인, 비밀번호 규칙 검증
- 로그인 성공 시 토큰 발급, 가입 완료 시 자동 로그인
- 회원 전용 기능(구매 등)에서 넘어온 경우, 로그인 후 원래 페이지로 복귀
- 로그인 실패 통합 안내 메시지 (계정 존재 여부 비노출)
- 계정 유형 구분: 일반 회원 / 판매자 — 판매자 계정은 판매자 페이지 접근 가능

---

## 2. 메인 (홈)

**목적**
- 우리 쇼핑몰이 "무엇을 파는 곳인지" 한눈에 보여주고, AI 채팅으로 자연스럽게 진입시키는 랜딩

**핵심 기능**
- AI 채팅 입력창: 질문 입력 시 챗봇 화면으로 전환, 입력 내용이 첫 메시지로 전송
- 카테고리 목록: 클릭 시 해당 카테고리명이 채팅 메시지로 입력되며 채팅 화면으로 이동
- 추천 상품 섹션: 로그인 사용자는 "OO님을 위한 추천" (대화에서 추출한 프로필 기반), 비로그인은 인기 상품으로 대체
- 예시 질문 칩: 클릭 시 해당 문장으로 채팅 시작 (첫 방문자에게 사용법 안내 역할)

---

## 3. 챗봇 (검색 결과)

**목적**
- 서비스의 핵심. 자연어로 상품을 찾거나("무선 키보드 추천") 상황을 설명하면("자취 시작해") AI가 상품을 찾아주는 화면

**핵심 기능**
- 멀티턴 대화: 이전 맥락 유지 ("더 싼 걸로", "방수되는 걸로" 등 후속 조건 반영)
- 자연어 조건 필터링: 발화에 포함된 조건(소재, 가격대, 색상, 기능 등)을 추출해 상품 검색에 반영 (예: "린넨 소재 이불 추천해줘" → 이불 중 린넨 소재만 필터링)
- 구조화된 응답: 텍스트 + 상품 카드(상품명, 가격, 이미지, 추천 이유 한 줄) → 카드 클릭 시 상품 상세 이동
- 상황 기반 추천은 카테고리별 묶음으로 그룹핑해 표시
- 조건 칩: AI가 발화에서 추출한 조건(예: "기념일", "호텔 레스토랑", "원피스")을 결과 상단에 칩으로 표시. 칩의 X를 누르면 해당 조건이 제거된 재추천이 이뤄짐 (제거는 후속 메시지로 전달, 별도 API 없음)
- 대화 저장 정책: 대화 내용은 DB에 저장하지 않음 → 과거 대화 조회 기능 없음. 멀티턴 맥락은 현재 세션(sessionId, 10분 sliding TTL) 동안만 유지
- 대화 기반 프로필 업데이트: 정해진 저장 시점에 대화에서 선호 정보(관심 카테고리, 가격대, 취향 등)를 추출해 사용자 프로필을 갱신하고, 이를 홈 개인화 추천에 활용
- 챗봇 장바구니 담기: 대화 중 "저거 장바구니에 담아줘" 요청 시 AI가 상품을 장바구니에 추가하고 결과를 답변으로 확인. 게스트는 로그인 유도 응답, 옵션 누락 시 옵션 목록과 함께 되물음
- 응답은 SSE 스트리밍으로 표시(텍스트 실시간 출력), 응답 실패 시 재시도, 결과 없을 때 조건 완화 제안. 게스트도 횟수 제한 없이 사용 가능(개인화만 미적용)

---

## 4. 상품 상세

**목적**
- 구매 판단에 필요한 정보를 제공하고 장바구니/구매로 연결하는 페이지

**핵심 기능**
- 상품 이미지(복수), 상품명, 가격/할인율, 옵션·수량 선택
- 장바구니 담기 / 바로 구매 (비로그인 시 구매는 로그인 유도)
- 주요 특징 요약 + 스펙 표
- 리뷰: 평점 통계, 리뷰 목록, 후기 신고 (신고 건은 관리자 페이지에서 처리)
- 연관 추천 2종: 함께 구매하면 좋은 상품 / Jarvis 추천 대체 상품
- 브랜드 정보 영역 → 브랜드 홈으로 이동

---

## 5. 브랜드 홈

**목적**
- 특정 브랜드의 소개와 상품을 모아 보는 페이지. 상품 상세의 브랜드 영역에서 진입

**핵심 기능**
- 브랜드 소개 (이름, 로고, 설명)
- 해당 브랜드 상품 목록 (정렬: 인기순/최신순/가격순)
- 상품 카드 클릭 시 상품 상세 이동
- (있으면 좋음) 브랜드 내 카테고리 필터

---

## 6. 장바구니

**목적**
- 구매 후보 상품을 모아 검토하고 주문으로 넘어가는 페이지

**핵심 기능**
- 담긴 상품 목록: 개별/전체 선택, 수량 변경, 삭제
- 선택 상품 기준 금액 합산 (상품 금액 / 할인 / 총액)
- 선택 상품 주문하기 → 결제 화면 이동
- "이 상품들과 함께 구매하면 좋아요" 연관 추천
- 비어 있을 때: 안내 + 추천 상품 or 채팅으로 유도

---

## 7. 결제 화면

**목적**
- 주문 정보를 확인하고 모의 결제로 주문을 확정하는 페이지

**핵심 기능**
- 주문 상품 목록 확인
- 배송지 입력/선택 (저장된 배송지 불러오기)
- 결제 수단 선택 (모의 결제 — 실제 PG 미연동)
- 최종 금액 확인 → 결제하기 → 주문 완료 처리
- 결제 실패(모의 실패 케이스 포함) 시 안내 및 재시도

---

## 8. 마이페이지

**목적**
- 내 주문·활동·계정 정보를 관리하는 허브. 좌측 메뉴 기반 하위 페이지 구조

**하위 메뉴별 핵심 기능**

| 메뉴 | 핵심 기능 |
|---|---|
| 주문 내역 | 주문 목록/상세 조회 (주문 건별 배송 상태 값 표시 — 별도 배송 조회 페이지 없음), 후기 작성·조회, 환불/반품/교환 신청 |
| 취소·반품·교환 내역 | 신청 건 목록 및 처리 상태 조회 |
| 최근 본 상품 | 조회한 상품 목록 (최신순) |
| 찜 | 찜한 상품 목록, 찜 해제, 장바구니 담기 |
| 배송지 관리 | 배송지 추가·수정·삭제·기본 배송지 설정 (닉네임/비밀번호 변경 기능 없음) |
| 문의 내역 | 문의 챗봇(9번 페이지)에서 접수한 문의 목록 조회 — 읽기 전용 |

---

## 9. 문의 챗봇 (AI 고객센터)

**목적**
- 주문/배송/환불 등 서비스 관련 질문을 AI가 1차 응대하는 페이지 (상품 추천 챗봇과는 별개)

**핵심 기능**
- AI 문의 챗봇: 자주 묻는 질문 답변 (환불/반품/교환 절차, 서비스 이용 방법 등)
- 내 주문 연동 답변: 로그인 사용자의 "내 주문 어떻게 됐어?" 질문에 주문 상태 값 기반으로 답변
- 관련 페이지 연결: 답변에서 환불/반품/교환 신청, 주문 내역 등 해당 페이지로 이동 유도
- 문의 접수: 챗봇으로 해결되지 않는 내용은 문의로 남기고, 마이페이지 문의 내역에서 조회 (읽기 전용)
- 비로그인 사용자: 일반 안내는 가능, 주문 관련 질문은 로그인 유도

---

## 10. 판매자 페이지

**목적**
- 판매자(브랜드)가 자사 상품의 판매·고객 데이터를 확인하고, AI 에이전트에게 원인 분석을 받고, 상품 상세 페이지를 관리하는 페이지
- 판매자 권한 계정으로만 접근 (일반 회원과 계정 구분)

**핵심 기능**
- 자사 로그/지표 조회: 매출, 주문 내역, 조회수/담김수 등 자사 상품 데이터
- AI 분석 기능
  - 매출 이상 분석 (이상 징후 감지)
  - 매출 감소 이유 비교 (기간별/상품별 비교)
  - 고객 행동 분석
  - 이탈 원인 분석
- 판매자 에이전트 챗봇: "매출이 왜 떨어졌어?" 같은 질문에 자사 데이터 기반으로 답변, 자사 상품의 원인 분석 제공
- 상품 상세 페이지 수정
- (범위 외) 환불/반품/교환 처리, 정산, 문의 답변 — 판매자 페이지에서 다루지 않음

---

## 11. 관리자 페이지 — 문의 및 신고 관리

**목적**
- 운영자가 사용자 문의와 신고를 확인·처리하는 페이지 (관리자 권한 계정 전용)

**핵심 기능**
- 문의 관리: 접수된 문의 목록/상세 조회, 답변 등록 → 답변은 사용자 마이페이지 문의 내역에서 확인
- 신고 관리: 후기(리뷰) 신고 목록/상세 조회, 처리 (신고된 후기 숨김/삭제)
- 처리 상태 관리: 미처리 / 처리중 / 완료 구분
